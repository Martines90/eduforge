## 4. üéØ MATHEMATICAL THINKING / MINDSET

This domain represents the cognitive approach that makes mathematics powerful:

### Problem-Solving Mindset

**1. Abstraction**
- Translate messy reality into clean mathematical models
- Identify what's relevant and what can be ignored
- Use variables to represent unknown quantities
- Example: "How many people?" becomes variable *n*

**2. Systematic Approach**
- Break complex problems into manageable steps
- Solve one piece at a time, building toward solution
- Check each step before proceeding
- Example: Multi-step word problems require sequential reasoning

**3. Pattern Recognition**
- Identify mathematical structures in different contexts
- Recognize when two problems are mathematically similar
- Apply known solutions to new situations
- Example: Recognizing exponential growth in populations, finance, epidemics

**4. Logical Reasoning**
- Follow deductive chains: if A, then B; if B, then C; therefore if A, then C
- Identify assumptions and constraints
- Verify solutions make logical sense
- Example: Checking if answer is physically possible (negative time? impossible!)

**5. Estimation & Sanity Checks**
- Approximate before calculating exactly
- Verify final answer is reasonable
- Use dimensional analysis (units must match!)
- Example: "Does this building really need 10,000 tons of concrete?" ‚Üí Check calculation!

**6. Generalization**
- Move from specific cases to general principles
- Recognize when a solution method applies broadly
- Create formulas that work for any input
- Example: Not just "this triangle," but "all right triangles"

**7. Multiple Representations**
- View problems through different lenses: equation, graph, table, diagram
- Translate between representations
- Choose the most useful representation for the task
- Example: Function can be equation, graph, or table of values

**8. Precision & Rigor**
- Define terms clearly
- State assumptions explicitly
- Show work step-by-step
- Round appropriately based on context
- Example: Engineering requires significant figures awareness

**9. Optimization Thinking**
- Recognize when seeking maximum or minimum
- Identify constraints and trade-offs
- Balance competing objectives
- Example: Maximize strength while minimizing weight/cost

**10. Modeling & Approximation**
- Understand models are simplifications of reality
- Know when approximations are acceptable
- Recognize model limitations
- Example: Assuming constant acceleration when it actually varies slightly

### Mathematical Communication

**Clarity:**
- Express mathematical ideas clearly in words, symbols, diagrams
- Show reasoning, not just answers
- Explain "why" not just "what"

**Notation:**
- Use standard mathematical notation correctly
- Define variables before using them
- Write units with numerical answers

**Justification:**
- Support conclusions with mathematical evidence
- Reference formulas, theorems, or principles used
- Explain reasoning at each step
